My First Code Sprint

After our first week of coding, we were given a three hour code sprint. We were given design files, two html files, and two css files. We then had to alter the code of the files as provided to make the resulting web pages for the site look as similar to the design files as possible.

I went into the sprint nervous. It had been a hard, long week packed full of new information, and I had struggled a few times. However, when it came down to it, the code flowed quickly and elegantly. After I finished, I even had time to make the buttons on the "About" page spin when they were hovered over. Here is the gitHub page for my project. Enjoy!

Resizing for Mobile

The next week, we drove right in to responsive design. We learned and practiced resizing the CSS to make web pages adapt to smaller screen sizes (such as tablets and phones). It was intriguing watching the layout of the pages alter depending on the width of the screen

We took a page we had been working on throughout our first week, and had to alter the code to make certain that the web pages would display properly on both desktop and mobile devices. I struggled, knowing I was overlooking one thing in my code. After I found out what it was, I continued working to complete a stretch goal of creating a second page and making it responsive as well. The above photo is the index page sized for a tablet next to the same page sized for a phone. If you'd like to see my github page for this project, here it is!
